🛡️ DigiSafe

AI-Powered Real-Time Digital Arrest Scam Prevention for Senior Citizens

📌 Overview

DigiSafe is an Android-based security framework designed to protect senior citizens from psychological cyber scams such as Digital Arrest fraud.

Unlike traditional fraud systems that act after money is lost, DigiSafe intervenes during the manipulation phase and protects financial decisions in real time.

🎯 Key Features
1️⃣ Real-Time Suspicious Call Risk Engine (Offline)

Detects unknown long-duration calls

Identifies urgency spikes

Detects authority impersonation language

Measures repetition pressure

Identifies emotional dominance patterns

Detects continuous isolation behavior

2️⃣ 3-Level Smart Intervention System

On-screen fraud warning

Guardian alert activation

Forced intervention with banking lock suggestion

3️⃣ Cognitive Shield Mode

Activates when:

15+ minute continuous call

High urgency language

Emotional dominance detected

Actions:

Temporarily restricts banking apps

Displays law enforcement checklist

Offers police verification guidance

4️⃣ Guardian-Approved Large Transaction Protection

Guardian approval required for high-value transfers

Cooling-off delay mechanism

Risk-based transaction interruption

5️⃣ Smart Transfer Limit System

User-defined daily and per-transaction limits

Double confirmation for high-value transfers

6️⃣ Reverse Interrogation AI

Provides scripted counter-questions to disrupt scammer psychology.

7️⃣ Guardian Monitoring Dashboard

Risk logs

Suspicious call history

Transaction alerts

Location alerts

Evidence access

8️⃣ Automated High-Risk Call Evidence Vault

Securely stores call recordings preceding suspicious transactions

Enables export for legal reporting

🔐 Privacy First Design

Offline risk scoring

No cloud audio storage

Only alert notifications require internet

Local encrypted storage

Explicit user consent required

Android (Java/Kotlin)

Foreground Services

🧠 Innovation

DigiSafe focuses on psychological manipulation interruption rather than post-fraud reporting.

It introduces:

Real-time behavioral risk scoring

Guardian-linked transaction friction

Cognitive shield activation

Evidence preservation framework

🚧 Limitations

Uses rule-based detection (MVP)

Banking integration simulated via overlay

Requires accessibility permissions
